# gscore_miao-plugin 迁移路线

目标：尽量对齐 Yunzai 生态的 `miao-plugin`，在 GsCore 中提供相同或等价的命令体验、数据能力、图片渲染和配置项。

## 当前已完成

- 基础命令：帮助、版本、更新日志、设置。
- WebUI 配置：面板源、Miao/Enka/米游社/Mgg/Hutao、缓存、渲染、权限等。
- 面板数据源：Miao API、Enka、米游社、MiniGG、Hutao 的基础接入。
- Enka 角色详情解析：等级、命座、天赋、武器、圣遗物、双暴、充能等。
- 图片面板：总面板图、单角色面板图。
- 角色别名：常用原神角色别名。
- 圣遗物评分：已读取本地 Yunzai `miao-plugin/resources/meta-gs/artifact/artis-mark.js` 角色权重，并迁移绝缘4、西风武器、薙草高精等常见动态规则。
- 圣遗物图片：已新增单角色圣遗物详情图与圣遗物列表图，开始对齐原版 `character/artis-mark`、`character/artis-list` 的功能入口。
- 面板列表图：`喵喵面板列表 [UID]` 已优先输出图片列表；单角色面板图高度改为按内容动态裁剪，避免底部内容截断。
- 登录与签到：已新增米游社 Cookie 登录、查看/删除登录、原神每日签到入口，作为原版登录体系的第一阶段迁移。
- 伤害计算：首版通用估算。
- miao-plugin 风格自然命令：`喵喵雷神面板 <UID>`、`喵喵雷神圣遗物 <UID>`、`喵喵雷神伤害 <UID>`。
- UID 绑定：`喵喵设置uid <UID>` 后，面板/圣遗物/伤害/自然命令可省略 UID。
- 面板管理入口：`喵喵面板列表 [UID]`、`喵喵更新面板 [UID]`、`喵喵删除面板/解绑UID`。
- 单角色面板图已改为接近 Yunzai `profile-detail` 的竖版结构，并可复用本地 Yunzai `miao-plugin/resources` 中的角色立绘、武器图标、圣遗物图标。
- 补充 Enka 角色数字 ID 到中文名兜底映射，圣遗物槽位由数字占位改为花/羽/沙/杯/冠，常见词条 ID 已中文化。
- Miao/Mgg/Hutao/米游社返回的 `avatars` 已统一转换到 `characters` 详情模型，图片、圣遗物、伤害功能可复用。
- 图片渲染失败时会回退文本摘要，避免命令无响应。

## 仍需迁移

### 1. 命令兼容

- `#雷神面板`、`#雷神伤害`、`#雷神圣遗物` 等 Yunzai 原始无 `喵喵` 前缀命令的 GsCore 兼容策略。
- `#面板列表` / `#面板角色列表`。
- `#更新面板` / `#全部面板更新`。
- `#删除面板` / `#重载面板`。
- `#圣遗物列表`：已支持 `喵喵圣遗物列表 [UID]`，仍需继续补齐无前缀兼容。
- `#面板列表`：已支持 `喵喵面板列表 [UID]` 图片列表，后续继续对齐原版 ProfileList 样式和更多管理入口。
- `#登录` / `#签到`：已支持 Cookie 登录与每日签到；后续继续补齐扫码登录、自动签到、失效 Cookie 清理与私聊安全提示。
- `#最强雷神` / `#雷神排名` / 群排行相关命令。
- `#敌人等级` 与伤害计算参数命令。

### 2. 数据模型

- 继续根据真实 Miao/Mgg/Hutao/米游社响应补齐更多字段映射与名称映射。
- 扩展本地 UID 绑定为多 UID 管理、当前 UID 切换与跨群/跨 bot 迁移。
- 增加面板缓存持久化，支持多 UID、多角色面板列表。
- 增加 Enka textMap/名称映射，减少 hash 或未知名称。

### 3. 圣遗物评分

- 继续补齐更多 `character/<角色>/artis.js` 的角色专属动态规则。
- 继续贴近 `miao-plugin` 的 `posMaxMark`、主词条满分和词条收益细节。
- 继续美化圣遗物详情评分图，补齐原版 `posMaxMark`、分词条颜色与套装汇总展示。

### 4. 伤害计算

- 迁移逐角色伤害模板。
- 支持命座、天赋、武器、圣遗物套装、反应、增伤、抗性、防御区。
- 支持 `伤害1/伤害2/...` 模式与可配置敌人等级。
- 支持组队 Buff 与开关配置。

### 5. 图片与素材

- 迁移角色面板 HTML/CSS 截图适配层，实现与原版 `profile-detail.html/css` 更一致的渲染效果。
- 当前 Pillow 渲染层已复用原版素材，但还不是直接运行原版 HTML/CSS 模板。
- 支持普通/彩蛋立绘、随机立绘、自定义上传图片。
- 支持圣遗物详情图、伤害图、排行榜图、角色卡片图。
- 增加素材更新命令或资源同步说明。

### 6. 角色与 Wiki

- 角色卡片、老婆/老公查询。
- 天赋、命座、角色资料 Wiki。
- 今日材料、日历等 Wiki 功能。

### 7. 统计与排行

- 面板练度统计。
- 群排行、最强角色、最高分排名。
- 胡桃数据库统计、深渊/幻想真境相关统计。

### 8. 抽卡与其他功能

- 抽卡记录、抽卡统计、模拟抽卡。
- 表情/哈哈等非面板附属功能。

## 推荐迁移顺序

1. 优先补齐自然命令和 UID 绑定，让日常用法接近原版。
2. 统一所有数据源的 `characters` 模型，保证后续渲染/评分/伤害都能复用。
3. 迁移圣遗物评分权重与圣遗物详情图。
4. 迁移逐角色伤害模板与敌人等级/组队配置。
5. 迁移面板列表、更新、删除、排行。
6. 最后迁移 Wiki、角色卡片、抽卡与其他附属功能。
